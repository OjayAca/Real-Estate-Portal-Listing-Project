In the real estate listing portal, ensure the following role-based access control for property status updates:

Admin Role:
Admins can set the status for any property, including properties listed by agents.
Admins can set the following property statuses: Available, Inactive, Draft, Rented, and Sold for all properties across the portal.
Admins are responsible for reviewing and approving listings, especially marking properties as Inactive or Draft.
Agent Role:
Agents can only set the status of the properties they have listed.
Agents can set the following statuses for properties they manage: Available, Rented, and Sold.
Agents can mark properties as Draft when preparing them to be listed.
Agents can only mark properties they manage as Inactive in certain situations, but this may require admin approval.

Ensure the system enforces the following:

Only Admins can set a property status to Inactive or Draft across the entire platform.
Agents cannot alter the status of properties they do not manage.
Admins have oversight over property status changes to ensure compliance with business rules.
